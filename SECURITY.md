## Reporting a Vulnerability

We take security vulnerabilities seriously. If you discover a security issue, please report it responsibly.

### How to Report

1. **Do not** create a public GitHub issue for security vulnerabilities
2. Send an email with details of the vulnerability
3. Include the following information:
   - Description of the vulnerability
   - Steps to reproduce the issue
   - Potential impact
   - Any suggested fixes (optional)

### What to Expect

- **Initial Response**: We will acknowledge receipt of your report within 48 hours
- **Updates**: We will provide status updates every 5 business days
- **Resolution Timeline**: We aim to resolve critical vulnerabilities within 30 days

### After Reporting

- If the vulnerability is accepted, we will work on a fix and coordinate disclosure with you
- If the vulnerability is declined, we will provide an explanation
- Credit will be given to reporters who follow responsible disclosure practices

### Scope

This security policy applies to the ProfMatch frontend application. For backend API security concerns, please refer to the backend repository's security policy.

## Security Best Practices

When contributing to this project, please ensure:

- Dependencies are kept up to date
- No sensitive data (API keys, credentials) is committed to the repository
- User input is properly validated and sanitized
- HTTPS is used for all API communications
